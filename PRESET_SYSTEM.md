# Tiny Glade Mesh Presets System

## Overview
The enhanced export operator now includes an intelligent preset browser that helps users understand which attributes are required for each shader type. The system:

1. **Loads pre-parsed JSON** (`nani_meshes_parsed.json`) generated by the one-shot parser script
2. **Displays preset information** in the export dialog showing required vs optional attributes  
3. **Shows sample meshes** for each shader type to help understand what uses each preset
4. **Fast loading** - no complex parsing at runtime

## Setup

### 1. Generate the JSON File (One-time)
First, run the one-shot parser script to generate the JSON file:

```bash
python parse_nani_meshes.py "C:\Program Files (x86)\Steam\steamapps\common\Tiny Glade\assets\nani_meshes.ron"
```

This creates `nani_meshes_parsed.json` in the same directory.

### 2. Configure Addon Preferences
In Blender:
- **Edit вҶ’ Preferences вҶ’ Add-ons**
- Search for "Tiny Glade Import/Export"
- Expand the addon and configure:
  - **Presets JSON Path**: Point to `nani_meshes_parsed.json`
    - Default: `C:\Program Files (x86)\Steam\steamapps\common\Tiny Glade\assets\nani_meshes_parsed.json`

### 2. Using the Preset System

When you open the **Export Tiny Glade JSON** dialog:

1. **Enable "Use Shader Preset"** (default: ON)
   - Shows the preset browser panel

2. **Browse Available Presets**
   - All presets appear as a list with mesh counts
   - Click/highlight a preset name to see details

3. **View Preset Details**
   - **Required attributes** (вң“) - These must be exported for the shader
   - **Optional attributes** (в—Ӣ) - Additional attributes you can choose
   - **Sample meshes** - Examples of assets using this preset

4. **Manual Attribute Selection**
   - Use checkboxes in "Export Options" section to select which attributes to include
   - The preset info is purely informational to guide your selection

5. **Export Without Presets** (Optional)
   - Uncheck "Use Shader Preset" for classic manual selection only

## How It Works

### Preset Structure
The system loads pre-parsed data from `nani_meshes_parsed.json`:

- **Preset Name**: Shader type (e.g., "SolidVertexColor", "SolidVertexColorWindow")
- **Required Attributes**: Must be exported for this shader
- **Optional Attributes**: Additional attributes to enhance quality
- **Mesh List**: All assets using this shader
- **Removed Attributes**: Attributes that don't apply to this shader

### Example Presets

**SolidVertexColor** (Plants, decorations, items)
```
вң“ Vertex_Position
вң“ Vertex_Normal  
вң“ Vertex_Color
в—Ӣ flags
в—Ӣ is_metal_part
```

**SolidVertexColorWindow** (Windows, door frames)
```
вң“ Vertex_Position
вң“ Vertex_Normal
в—Ӣ Vertex_UV
в—Ӣ flags
```

**VelocityOnly** (Dynamic objects like carts)
```
вң“ Vertex_Position
  (Other attributes removed to optimize)
```

**SolidVertexColorWindowGlass** (Window glass)
```
вң“ Vertex_Position
вң“ Vertex_Normal
вң“ Vertex_UV
вң“ is_glass
```

## Files Structure

### New Files
- **`mesh_presets.py`**
  - `MeshPresetManager` class - Core preset system
  - Parses `nani_meshes.ron` using regex
  - Maintains mesh вҶ’ preset mappings
  - Scans asset folder for verification

- **`addon_preferences.py`**
  - `TinyGladeAddonPreferences` class
  - Path configuration UI in addon preferences
  - Persists user settings

### Modified Files
- **`__init__.py`**
  - Registers addon preferences
  - Initializes preset manager on addon load
  - Creates scene properties for enum lists

- **`export_operator.py`**
  - Added preset selection properties
  - New preset browser UI in export dialog
  - Shows required vs optional attributes

## Troubleshooting

### "No presets loaded"
- Verify addon preferences paths are correct
- Ensure nani_meshes.ron exists at the configured path
- Restart Blender after changing preferences
- Check console for error messages

### Presets show but meshes/attributes are empty
- Verify nani_meshes.ron file is not corrupted
- Check file permissions (read access)
- Restart Blender to force reload

### Wrong attributes showing for a mesh
- Update nani_meshes.ron if game assets changed in-game
- Addon only reloads presets on Blender restart

### Cannot select presets in export dialog
- The preset browser is informational only
- Use checkboxes in "Export Options" to select attributes
- Presets guide your manual selection

## Advanced Usage

### To update for new game content:
1. Get new `nani_meshes.ron` from the game
2. Place it at configured path
3. Restart Blender to reload presets

### To customize preset requirements:
1. Edit `nani_meshes.ron` entries to add/remove attributes
2. Restart Blender
3. Presets will reflect your changes

## Performance Notes
- Preset loading happens once on addon startup
- Parsing typically takes <100ms
- Asset folder scan limited to 100 files for performance
- In-memory caching prevents repeated parsing
